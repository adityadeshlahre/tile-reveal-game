# tile-reveal-game

## Stack

- **Astro 6** + **`@astrojs/cloudflare`** adapter — SSR on Cloudflare Workers.
- **Bun** (package manager; `bun.lock` present, never `npm`/`pnpm`).
- **Node >= 22.12.0** required.
- **Vite ^7** (enforced via `overrides` in `package.json`).
- **Tailwind v4** skill configured (`.claude/skills/tailwind-4-docs/`).
- No test framework, no linter, no formatter — none are installed.

## Commands

| Action              | Command                                                                      |
| ------------------- | ---------------------------------------------------------------------------- |
| Dev server          | `bun dev` (localhost:4321)                                                   |
| Build               | `bun build` → `dist/`                                                        |
| Preview             | `bun preview` (builds first)                                                 |
| Deploy              | `bun deploy` (build + `wrangler deploy`)                                     |
| Regenerate CF types | `bun generate-types` (alias: `bun cf-typegen`) → `worker-configuration.d.ts` |

All scripts are defined in `package.json`. Run `bun astro` to pass subcommands to the Astro CLI.

## Context discovery

Always read `DESIGN.md` first for the visual system. Then browse `.agents/skills/` and root `*.md` files for instructions and design reference not encoded elsewhere.

## Key files

- `wrangler.jsonc` — Worker config; `compatibility_date`, `observability.enabled`, `global_fetch_strictly_public` compat flag.
- `astro.config.mjs` — Astro config; only sets the Cloudflare adapter.
- `worker-configuration.d.ts` — Generated by `wrangler types`. Do not edit manually.
- `src/env.d.ts` — Astro+Cloudflare runtime types (`App.Locals` extends `Runtime<Env>`).
- **Workers AI binding** accessible as `Astro.locals.env.AI` in Astro components (SSR only).
- `DESIGN.md` — Design system reference (Clay.com-inspired). Contains token definitions for colors, typography, spacing, and components. Source of truth for visual language.

## Architecture

- Early-stage project; current source is the default Astro starter template (`src/pages/index.astro`, `src/layouts/Layout.astro`, `src/components/Welcome.astro`).
- `src/pages/` — Astro routing (file-based).
- `src/components/` — Astro components.
- `dist/` — build output (gitignored).
- `.astro/` — generated types (gitignored).
- `.wrangler/` — local Cloudflare state (gitignored).

## Generated code

- `worker-configuration.d.ts` is regenerated via `bun generate-types`. Run after changing Wrangler bindings.
- `.astro/types.d.ts` is auto-generated by Astro. The `tsconfig.json` includes it.
